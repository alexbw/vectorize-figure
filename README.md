# Vectorize Figure

Generative image models (Nano Banana, gpt-image, and the like) draw beautiful scientific figures. The trouble is the data is fiction. The axes are painted on, the curve is a guess, the error bars mean nothing, and you cannot edit any of it without re-prompting and hoping. You get a 2048-pixel JPEG where you wanted a plot.

This is a skill that teaches a coding agent to take one of those raster figures and rebuild it as a semantic JSON spec plus an HTML file that renders the figure from that spec. Every axis, tick, label, legend, and data mark becomes an editable field. You swap in your real numbers, change a domain or a color, and the figure redraws. The source image is never reused as a visual layer — the reconstruction is drawn from scratch, so what you ship is structure, not pixels.

## Before and after

`assets/reference/reference-01-place-code-opto-A-reference.png` is an AI-generated panel: two place-cell heatmaps with a colorbar. Point the skill at it and you get `outputs/reference-01-place-code-opto-A-vectorize-figure/`:

- `reference-01-place-code-opto-A.json` — the figure as data: canvas dimensions, a viridis colormap with stop values, panel geometry, axis lines, tick coordinates, labels, and the recipes that generate the heatmap values.
- `reference-01-place-code-opto-A.html` — a renderer that reads the JSON and draws the panel with canvas for the heatmap and SVG/DOM for axes and text.

Edit the JSON, reload the HTML, and the panel changes. No prompt, no model, no pixels copied from the original.

## Quickstart

The skill is a markdown file. It works with any agent that reads instructions from a file — Claude Code, Codex, Cursor, and others.

1. Copy `skills/vectorize-figure/` into your agent's skills directory (for Claude Code, `~/.claude/skills/`), or just point your agent at `skills/vectorize-figure/SKILL.md`.
2. Give it an image and ask for a reconstruction:

   > Reconstruct this figure into editable JSON and HTML using the vectorize-figure skill: `my-figure.png`

3. Open the generated HTML in a browser. Edit the JSON to put your real data in.

The skill bundle (`SKILL.md`, the three files in `references/`, and `assets/hybrid-renderer-template.html`) is self-contained. The `scripts/` and `schemas/` helpers in this repo are for validating the bundled examples and assume you are running inside this checkout.

## How it works

The skill rests on one rule and one representation.

The rule: the generated candidate must not display the source raster. No `<img>`, no `drawImage`, no `data:image`, no CSS background pointing at the original. If the figure shows up, it has to be redrawn from the spec. Source images are allowed only in clearly labeled QA views that sit outside the generated surface. This is what forces a real reconstruction instead of a screenshot with a caption.

The representation is a semantic intermediate form: panels, plot boxes, axes with domains and source-calibrated tick coordinates, marks, annotations, legends, colorbars, style tokens, and provenance notes that record what was measured versus inferred. Editable structure (axes, ticks, labels, paths, legends) renders as SVG or DOM. Dense generated fields (heatmaps, rasters, large scatter) render on canvas. The IR design and the field contract live in `skills/vectorize-figure/references/`.

## Examples and local viewing

Serve the repo and open the viewers:

```bash
python3 -m http.server 8765
```

- `http://localhost:8765/outputs/` — inspection hub linking every generated viewer
- `http://localhost:8765/outputs/full-figure-batch-viewer.html` — full composites, generated vs. reference toggle
- `http://localhost:8765/outputs/vectorize-figure-batch-gallery.html` — 62 cropped subpanels, generated vs. reference toggle
- `http://localhost:8765/examples/no-image-1c/index.html` — a single panel reconstruction with a QA-only reference toggle
- `http://localhost:8765/examples/no-image-four/index.html` — a four-panel reconstruction built from synthetic data, no reference image at all

## Reference images

Every image under `assets/` was generated by an AI image model prompted to produce publication-style figures. None come from a published paper, and the data they depict is hallucinated by construction — that is the problem this project exists to solve. They double as the test fixtures: the output specs store pixel coordinates calibrated against these exact files, so the images must not be resized or recompressed. See `assets/README.md`.

## Validation

Python checks need Pillow; browser checks need Chrome.

```bash
python3 -m pip install -r requirements.txt
```

The workflow runner compiles the harness, syntax-checks the renderer's inline JavaScript, runs the validator unit tests, audits semantic IR field coverage against `schemas/ir-field-coverage.json`, validates every output spec, and screenshots the generated surfaces to confirm no source raster leaks in.

```bash
python3 scripts/validate_vectorize_figure_workflow.py --jobs 4
python3 scripts/validate_vectorize_figure_workflow.py --all-variants --skip-browser --jobs 4
python3 scripts/validate_vectorize_figure_workflow.py --all-gallery --jobs 4
```

Set `CHROME_BIN` if Chrome is not on the default path, or use `--skip-browser` for structural-only runs.
